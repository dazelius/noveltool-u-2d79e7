# 실명·실지명 노출 금지 — 마스터 룰

이 인스타툰의 모든 집필·연출 결정에서 **최우선으로 지켜야 하는 절대 규칙**.

## 원칙

이 작품은 작가 본인 가족의 일상을 모티프로 한 인스타툰이다. 그렇기에 **공개되는 콘텐츠(이미지·텍스트) 안에 실제 인물의 본명이나 실제 가게·지역의 상호가 노출되어서는 안 된다.**

볼트 내부의 **slug · title · 본문(작가 노트)** 은 작업 편의를 위해 본명·실지명을 그대로 써도 무방하지만, **독자에게 도달하는 모든 표면**에서는 가공의 호칭/별칭만 노출된다.

## 적용 범위 (절대 금지)

다음 필드에 본명/실지명이 들어가서는 **안 된다**:

1. **storyboard panels 의 dialogue** (말풍선 텍스트)
2. **storyboard panels 의 narration** (내레이션 박스)
3. **storyboard panels 의 sfx** (효과음)
4. **storyboard panels 의 bubbles[].text** (overlay 말풍선)
5. **panels 의 action 묘사 중 "이미지 안에 그려질 글자" 지시** — 간판·표지판·메뉴판·핸드폰 화면 텍스트 등.

`action` 의 일반 산문 묘사("탑골순대국 4인 테이블에 앉아 있다") 는 작가/AI 용 메모이므로 본명·실지명을 그대로 써도 되지만, **그 묘사가 "간판에 ○○ 글자" 식으로 이미지에 글자를 박으라고 지시한다면** 그 글자는 가공명이어야 한다.

## 호칭 변환 표

| 실명 (볼트 내부) | 작품 표면에 노출 가능한 호칭 |
|---|---|
| 이두형 | "두형이"는 작가 자칭 닉네임이라 OK. 그러나 풀네임 "이두형" 은 노출 금지. 호칭은 "여보", "여보야", "아빠", "오빠" 권장. |
| 오은주 | 풀네임 "오은주" 는 노출 금지. "여보", "엄마", "자기", 두형이 입장에선 "누나" (한 살 누나) 권장. 별칭 "은주씨" 도 가급적 회피. |
| 이재윤 | "재윤" / "재윤아" / "재윤이" 모두 노출 금지. 항상 태명 **"밤톨이/밤톨아"** 로. |
| 이채은 | "채은" / "채은아" / "채은이" 모두 노출 금지. 항상 태명 **"곶감이/곶감아"** 로. |
| 관리사님 | 본명 미지정. "관리사님" / "이모님" 그대로 OK. |

> 쌍둥이는 태명이 이미 캐릭터 컬러로 자리잡았으므로 본편 dialogue 에서 "재윤아—" 식으로 부르지 말고 **항상 "밤톨아—" / "곶감아—"**.

## 지명 변환 표

| 실지명 (볼트 내부 slug) | 작품 표면에 노출 가능한 표기 |
|---|---|
| world/수지구청 | "수지" "수지구청" "용인" 등 노출 금지. "동네", "우리 동네", "구청 앞" 같은 일반명사로. |
| world/탑골순대국 | "탑골순대국" "탑골" 노출 금지. "동네 순대국집", "단골 순대국집", "그 집" 으로. |
| world/수지-코인노래방 | "수지 코노" 노출 금지. "코인노래방", "코노", "근처 코노" 는 일반명사라 OK. |

## 체크리스트 (집필 시 매번 확인)

새 panel 을 쓰거나 수정할 때 dialogue / narration / sfx / bubbles 를 작성한 뒤 **반드시 다음을 자체 검수**:

- [ ] 풀네임 "이두형 / 오은주 / 이재윤 / 이채은" 이 들어갔는가? → 있으면 호칭으로 교체.
- [ ] 쌍둥이를 "재윤" / "채은" 으로 불렀는가? → "밤톨" / "곶감" 으로 교체.
- [ ] "수지" / "탑골" 같은 실지명이 노출됐는가? → 일반명사로 교체.
- [ ] action 안에 "○○ 간판이 보인다" 식의 글자 burn-in 지시가 있는가? → 글자 자체는 가공명이거나, 글자 없이 실루엣만.

## 이미 burn-in 된 컷의 처리

이미 생성된 이미지에 실명·실지명이 박혀버린 경우, **데이터(panels)부터 먼저 가공명으로 수정**한다. 이미지 재생성은 작가의 별도 결정에 따른다 — 데이터가 깨끗하면 다음 재생성 때 자동으로 가공명 버전이 그려진다.
